Detailed Description

The OmniLog is an adapter class for invoking events and sending them to a log collector.

The underlying implementation may be using a CDP or CommonLog.

The objective of the class is to provide easy-to-use interface for developers to send events, while at the same time retaining and using stateful context to enrich the event with data, such as session id, session channel information, experience id, flow id, etc.
